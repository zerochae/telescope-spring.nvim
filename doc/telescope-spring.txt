5. USAGE                                             *telescope-spring-usage*

The plugin provides multiple ways to access Spring Boot endpoints:

Simple Commands (Recommended):~
    :Spring Get      " Find all GET endpoints
    :Spring Post     " Find all POST endpoints
    :Spring Put      " Find all PUT endpoints
    :Spring Delete   " Find all DELETE endpoints
    :Spring Patch    " Find all PATCH endpoints

Alternative Commands:~
    :SpringGetMapping     " Find all GET endpoints
    :SpringPostMapping    " Find all POST endpoints
    :SpringPutMapping     " Find all PUT endpoints
    :SpringDeleteMapping  " Find all DELETE endpoints
    :SpringPatchMapping   " Find all PATCH endpoints

Cache Management Commands:~
    :Spring SaveCache     " Force save cache to disk (persistent mode)
    :Spring ClearCache    " Clear all cached data
    :Spring CacheStatus   " Show detailed cache information

Telescope Extension:~
    :Telescope spring        " Default picker (GET endpoints)
    :Telescope spring get    " GET endpoints
    :Telescope spring post   " POST endpoints
    :Telescope spring put    " PUT endpoints
    :Telescope spring delete " DELETE endpoints
    :Telescope spring patch  " PATCH endpoints

10. CACHING                                        *telescope-spring-caching*

The plugin includes an intelligent caching system with three modes:

Cache Modes:~

    "time" mode (default):
        Cache expires after the specified `cache_ttl` milliseconds.
        Useful for development when endpoints change frequently.
        
    "session" mode: 
        Cache remains valid until Neovim is closed.
        Useful for stable codebases or long coding sessions.
        
    "persistent" mode:
        Cache is saved to disk and persists across nvim sessions.
        Provides maximum performance for large projects.

Basic Configuration:~
>lua
    {
      cache_ttl = 10000,       -- Cache for 10 seconds (time/session modes)
      cache_mode = "time",     -- Use TTL-based caching
    }
    
    {
      cache_mode = "session",  -- Cache until nvim closes
    }
    
    {
      cache_mode = "persistent", -- Disk-persistent cache
      debug = false,           -- Enable for troubleshooting
    }
<

                                              *telescope-spring-persistent-cache*
Persistent Cache Mode:~

The persistent cache mode offers the best performance for large projects:

Features:
  â€¢ ğŸ“ Project-specific caching: Each project gets its own cache directory
  â€¢ ğŸ’¾ Disk storage: Cache survives nvim restarts and system reboots
  â€¢ ğŸš€ Instant loading: No re-scanning on subsequent launches  
  â€¢ ğŸ”„ Smart invalidation: Detects when annotations need re-scanning
  â€¢ ğŸ—‚ï¸ Cache location: `~/.local/share/nvim/telescope-spring/[project-name]/`

Cache Management Commands:~
    :SpringSaveCache      Force save current cache to disk
    :SpringClearCache     Clear all cache files for current project
    :SpringCacheStatus    Show detailed cache information
    
    :Spring SaveCache     Same as :SpringSaveCache
    :Spring ClearCache    Same as :SpringClearCache  
    :Spring CacheStatus   Same as :SpringCacheStatus

Cache Files Structure:~
>
    ~/.local/share/nvim/telescope-spring/my-project/
    â”œâ”€â”€ find_cache.lua    " Endpoint data (paths, methods, locations)
    â””â”€â”€ metadata.lua      " Scan history and project metadata
<

When to use persistent mode:
  âœ… Large Spring Boot projects with many controllers
  âœ… Frequent nvim restarts during development
  âœ… Want maximum performance after initial scan
  âŒ Small projects (overhead not worth it)
  âŒ Controllers change very frequently

The cache automatically invalidates when:
  â€¢ TTL expires (in "time" mode)
  â€¢ Neovim is restarted (in "session" mode)
  â€¢ Annotations are modified (in "persistent" mode)
  â€¢ Plugin is reloaded

12. TROUBLESHOOTING                        *telescope-spring-troubleshooting*

Common Issues:~

Q: Plugin doesn't work after installation
A: Make sure you called `require("spring").setup()` in your config function.
   The plugin requires explicit setup to function properly.

Q: No endpoints found
A: Check that:
   â€¢ You're in a Java/Spring Boot project
   â€¢ Java files contain @RequestMapping, @GetMapping, etc. annotations
   â€¢ `file_patterns` includes the correct file types
   â€¢ `exclude_patterns` isn't excluding your source files

Q: Cache not working as expected  
A: Verify your `cache_mode` and `cache_ttl` settings:
   â€¢ "time" mode: cache expires after TTL milliseconds
   â€¢ "session" mode: cache persists until nvim closes  
   â€¢ "persistent" mode: cache saved to disk, check :SpringCacheStatus

Q: Persistent cache showing incomplete results
A: Enable debug mode to troubleshoot:
   â€¢ Set `debug = true` in setup configuration
   â€¢ Use :SpringCacheStatus to check cache state
   â€¢ Try :SpringClearCache and re-scan if needed

Q: Icons not displaying
A: Ensure:
   â€¢ `show_icons = true` in ui configuration
   â€¢ Your terminal/font supports the configured emoji icons
   â€¢ Try different icon themes if needed

Q: Colors not working
A: Check that the highlight groups in `method_colors` exist in your colorscheme.
   Use `:highlight` command to verify available groups.

Debug Information:~
    Check if setup was called: `echo g:spring_setup_called`
    View current config: `lua print(vim.inspect(require("spring").get_config()))`
